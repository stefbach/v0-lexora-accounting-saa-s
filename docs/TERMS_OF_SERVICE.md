## 1. SERVICE DESCRIPTION

### 1.1 Overview

Lexora provides a cloud-based Software-as-a-Service (SaaS) accounting platform designed specifically for Mauritian businesses. The platform includes the following modules:

| Module | Description | Key Features |
|---|---|---|
| **General Ledger (GL)** | Central accounting records | Journal entries, trial balance, reporting |
| **Invoicing** | Customer & supplier invoice management | Creation, approval, payment tracking |
| **Bank Reconciliation** | Bank statement matching & OCR | Transaction classification, automated matching |
| **Payroll** | Salary calculation & tax withholding | PAYE, CSG, NSF, employee statements |
| **Tax Compliance** | MRA reporting & filing | PAYE declarations, VAT returns, income tax |
| **Reporting & Analytics** | Financial statements & dashboards | P&L, Balance Sheet, Cash Flow, KPIs |

### 1.2 Service Levels

**Availability Target**: 99.5% uptime (per month, measured at API level)

- Planned maintenance: 0-4 hours per month (announced 14 days in advance)
- Unplanned outages: Monitored continuously with 15-minute incident response time
- Data backup: Continuous (RPO = 1 hour, RTO = 4 hours for GL module)

**Support**:
- Email support: Response within 24 hours (business days)
- Critical incidents: 24/7 phone support (included for compliance-impacting issues)
- Feature requests: Reviewed quarterly

### 1.3 System Access

**User Roles & Permissions:**

| Role | Description | Permissions |
|---|---|---|
| **Propriétaire SaaS** | Company owner / administrator | Create users, manage access, view all data |
| **Directeur** | Finance director / CFO | Approve GL entries >10,000 MUR, reconciliation sign-off |
| **Comptable** | Staff accountant / bookkeeper | Create GL entries, reconcile bank, match invoices |
| **Agent RH** | HR staff | Manage employee data, calculate payroll (no approval) |
| **Agent Administratif** | Administration staff | View reports only, export data |
| **Auditeur Externe** | External auditor (read-only) | Access GL, invoices, bank data (no modification) |

**Access Control Mechanisms:**
- Password-based authentication (minimum 12 characters, special characters required)
- Multi-factor authentication (MFA) available (recommended for director roles)
- IP whitelisting (optional, for enhanced security)
- Session timeout (30 minutes of inactivity)
- Concurrent session limit (1 login per user, unless authorized)

### 5.3 Audit Trail Immutability

**All audit trail records are immutable (cannot be deleted or modified):**

Audit Logged:
- Who: User ID, email, role
- What: Action (created invoice, posted GL entry, matched transaction)
- When: Timestamp (accurate to 1 second)
- Where: IP address, device
- Why: Reference code (invoice ID, GL entry ID, bank transaction ID)

**GL Entry Modification Tracking:**
```
Example: GL entry #ECC-2026-00047 (originally posted as expense)

Initial creation:
├─ Created by: comptable@company.mu (2026-05-20 10:15:30 UTC)
├─ Action: Posted GL entry (2 lines: Debit 601, Credit 4020)
└─ Status: Posted (immutable, cannot be changed)

Discovered error (2026-05-22):
├─ Error: Should have been Account 602 (not 601)
├─ Solution: Cannot modify GL entry (immutable)
├─ Must create: Reversal entry (2026-05-22 14:00:00)
│  ├─ Reversal of ECC-2026-00047
│  ├─ Posted by: comptable@company.mu
│  └─ Approved by: directeur@company.mu
├─ Then create: Corrected entry (2026-05-22 14:05:00)
│  ├─ New GL entry with correct account 602
│  ├─ Posted by: comptable@company.mu
│  └─ Approved by: directeur@company.mu
└─ Audit trail shows: All 3 entries linked, complete history

Auditor can see: All changes, all approvals, complete justification
```

**Audit Trail Access for Auditors:**
- Dashboard → Audit → GL Entry History → ECC-2026-00047
- Shows: Creation, attempted modification (if any), reversal, correction
- Evidence: User names, timestamps, approval workflow
